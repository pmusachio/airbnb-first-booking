# Notebooks

Suggested reading order — these notebooks tell the analytical story of the
project, from problem framing to business translation, and are the best places
to grab portfolio screenshots:

1. `00_business_understanding.ipynb`: problem statement, business assumptions and analytical strategy.
2. `01_data_understanding.ipynb`: schema, missing values, samples and data quality checks.
3. `02_exploratory_analysis.ipynb`: distributions, hypothesis testing and storytelling charts.
4. `03_feature_engineering.ipynb`: domain-driven transformations and engineered features.
5. `04_modeling_and_business_results.ipynb`: metrics, artifacts and business interpretation.

The notebooks reuse the same package code from `src/`, so the analytical
narrative always stays in sync with the reproducible pipeline that powers the
CLI and the API.
